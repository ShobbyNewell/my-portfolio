My Masters of Engineering Experience

Here I go through a few courses I did during my Masters at the University of Toronto. I was in the Aerospace department, and most of my classes were at UTIAS. The reports I did for these courses can be found here, but I talk about a few of them in my own words, and how it has influenced me to pursue other related things.

AER 1403: Advanced Aerospace Structures

Project 1: Optimal design of composite sandwich beams

In this project, we were to design the length and width of a composite sandwich beam (composed of 2 stiff composite faces with a lightweight polymer core), with pre-determined material properties, and a given load, such that a minimum mass is reached.

Sandwich beams like this are subject to a few mechanisms of failure: elastic indentation, core shear, face microbuckling, and face wrinkling (though face wrinkling wasn't explored in this project), and we want to achieve a minimum mass index M(hat) given a structural load index P(hat). This is done through a failure mechanism map, that uses the face and core thickness ratios as axes, and plots the boundary regions where a specific failure is dominant. The optimal design trajectory rides the boundary between 2 failure mechanisms, and will give you the desired optimal length and width.

Given that sandwich beams and panels are used heavily in aerospace structures, finding a lowest mass solution made this project really relevant for economic and ecological pursuits. I hope that someday in the future, thin sandwich beams can make their way into small RC planes, as I'd love to experiment with them instead of conventional balsa wood.

Project 2: Finite Element Solution for a Curved Bracket

For this project, we were given a semi-circular bracket in 2D plane stress, and were asked to write finite element code, using isoparametric quadrilateral elements, to calculate displacements of the bracket. There's lots of tool libraries or commercial software to tackle this problem, such as ANSYS, but we were asked to write our own code (usually in MATLAB).

The process involved defining the geometry, and discretizing it using quadrilateral elements (each element with 4 nodes) to create a mesh. The geometry and displacement of these elements are interpolated with it's shape functions and their derivative, which are defined in a natural coordinate system (ξ,η). Basically, they show how a node in each element reacts with respect to the other. We can then use a Jacobian matrix to map from natural coordinates to Cartesian coordinates (x,y), and get the global shape function derivative. That is then used to build a strain displacement matrix B, and then you have enough to create an element stiffness matrix. This stiffness matrix relates the nodal displacements to the nodal forces, showing how deformed each element will be given a force on each of the nodes.

With the element stiffness matrix, we try to build a global stiffness matrix, that relates the entire structure's displacement to it's global force that is acting on it. This is done by numbering each node, going over each element, and scatter-adding each element stiffness matrix until we get the global stiffness matrix. The work to build this global stiffness matrix can be cut down if the shape is symmetric about one or more axis.

Next we apply boundary conditions (such as no displacements at the wall), and enter the global force (attached to a given node), to find the global displacement vector of the object. I also plotted some displacement contours and stress fields as part of the project. A cool aspect of this project is that this process can be repeated with a finer mesh, and the displacement can be compared. When the changes are minimal, you know you've achieved convergence.

This was one of the hardest projects I had to do, and while I can't see myself doing FEA from scratch like this in a professional setting, I can fully appreciate the backend work commercial software like ANSYS offers now.

AER1216: Fundamentals of UAV

Simulation Analysis of Fixed-Wing and Multi-Rotor Aircraft

For this project, we were to create a control autopilot system for a fixed-wing Aerosande UAV, which can receive navigational commands of airspeed, heading, and altitude, and compute the control surface and throttle commands. Additional calculations for range, endurance, and accounting for wind conditions were also tested. There was also a multirotor section, where we tested it's performance (range, endurance) and a linearized dynamics model was used for the multirotor to hover at a point and trace a flat horizontal circle.

The framework for these control systems were given on MATLAB's Simulink, and we got to tune them using PID controllers to test for stable response times and resistance to spikes from external variables, like a sudden change in wind. The whole idea behind this project was to automate a complex flight path or revert to a landing during an emergency, like if the aircraft goes out of range or experiences interference.

As an RC (remote control) enthusiast, this project opened my eyes onto flight controllers and FPV flying. Before this project, I was into basic scratch building RC planes out of foam, and flying them with a basic radio every now and then. This project opened me up to ArduPilot, software that runs on a flight controller and can access information from the electronics of the plane/multirotor (like the battery, motors, gyros, GPS, etc), and can create a flight plan based on it. While I've not fully mastered it yet, it has reignited a spark I used to have about the RC hobby when I was little, and I can't wait to experiment and learn more about it.

AER1217: Development of Autonomous UAS

Autonomous Drone Racing Competition

This project was very interesting: Develop an autonomous path planning algorithm for a Crazyflie quadrotor to takeoff, maneuver around a few obstacles, go through some gates, and land at a specified point. It started with a given Pybullet simulation environment, where we used path planning algorithms that are search based (A*), and sampling based (RRT/RRT*) to maneuver the drone. Once it was shown in Pybullet that the drone maneuvered correctly, we tried to decrease the time taken for this process, as after all, it was a race with other groups. We focused on tuning parameters such as step size, max iterations, and path rewriting range to prevent collision. After that, we tested a few a different gate sequences, as in the final competition those would be randomized.

There was a practice session where we were able to physically test the drone against our simulation, and we did pretty well on it, but in the actual competition, the drone only went through 4/6 gates. We encountered issues with controlling the drones speed, causing it to hit the gate, and only after the competition (but before the report submission) did we figure out why. In the simulation, the Crazyflie's command interface is given, with high level commands such as takeoff, land, and goTo, which we used for taking off. There's also the low level command, cmdFullState(), which we used for trajectory/path planning (going through the gates, and avoiding the obstacles). In the simulation, we used a control input iteration to change the drone's speed, but actually needed to change a time duration variable instead, because of the switching of control state from high level to low level. This was not evident in simulation, but when you combine the real world latency of the drone from the VICON system, the results were not the best. These problems were not evident in our practice session, as we only went through 4 gates, not 6, which minimized the swaying trajectory caused by the control input iteration speed technique.

Still, the experience of the competition was fantastic, and our group didn't feel too bad, as half of the other groups also couldn't complete the full sequence. I've built quadrotors before (assembling the frame and motors), but when it comes time to tuning the sensors and flight controller, I usually follow a pre-determined tutorial. After this project, it incentivized me to play around with tuning my own drone, and look into other Pybullet simulations.

AER 1516: Motion planning for Robotics

Analyzing and improving path planning algorithms

There were 2 projects for this course; the first was a presentation based on an academic paper, ours was on "Smooth coverage path planning and control of mobile robots based on high resolution grid map representation", authored by Tae-Kyeong Lee, Sang-Hoon Baek, Young-Ho Choi, and Se-Young Ho. This paper was about improving the efficiency of map coverage for a robot (like how a Roomba covers a dirty floor) by reducing path overlap and increasing the average velocity during coverage. It's important to motion planning as it aims to improve energy efficiency at a reduced cost. Essentially, the proposed method involved mapping out the space into a high resolution grid map composed of small cells, allowing the robot to have a higher degree of freedom. Then a path is planned where the robot navigates in the grid, avoiding obstacles and cells that have already been visited, and that path is improved using Bezier curves (parametric curves used to model smooth and scalable shapes). They also apply Particle Swam Optimization (PSO) technique, which minimizes the curvature of the path to allow the robot to maintain optimal velocity. Overall this project gave a great insight into path planning academic papers, and how algorithms are improved upon using existing methodologies.

The second project was improving upon an existing academic paper, ours was on "Analysis of Obstacle Avoidance Strategy for Dual-Arm Robot Based on Speed Field with Improved Artificial Potential Field Algorithm", by Zhang, H., Zhu, Y., Liu, X., & Xu, X. Essentially, the paper covered path planning of the arms of a dual arm robot, so that they don't collide with static or dynamic obstacles (including the other arm). They introduce an extended algorithm based on potential field velocity (VP-APF). The method integrates traditional APF attraction and repulsion functions with new concepts of speed, mapping these into the joint space using the Jacobian matrix to establish differential velocity functions for joint motion. We tested this VP-APF method in a MATLAB environment using a rigid body tree robot model, with new simulations planned in dynamic settings involving moving obstacles, and even made a video presentation that you can see. While it's not enough to write a whole academic paper on, this project gave me deeper respect to the academic authors who are always trying to improve existing algorithms.

MIE 1514: Systems Design and Engineering

Developing a new product: Ace Master Tennis Robot

The main project for this course was to design a new product or service using systems engineering, ours was on a tennis training robot that you can play against. A camera vision system would track your movements, and AR glasses would act as a user interface (displays score , user tutorials, etc). These 3 items (robot, camera, and AR glasses) interact with each other to form one system, and we wrote a whole report on it's design requirements, modeling & simulation, benchmarking, competitiveness in the market, system reliability, and more. While I have taken engineering design courses in my Bachelors, this course expanded on my knowledge in regards to troubleshooting problems within a product or system, assessing its life cycle, and leaving it open for continuous improvement.
